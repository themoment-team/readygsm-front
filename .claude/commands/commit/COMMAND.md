# /commit

변경된 파일을 분석해서 적절한 커밋 메시지를 작성하고 커밋합니다.

## 사용법

```
/commit
```

## 커밋 메시지 형식

`.claude/commands/commit/references/scope-guide.md` 를 참고해서 메시지를 작성하세요.

```
type(scope): 설명
```

- 설명은 **한국어**로 작성
- 명령형으로 작성 (예: "추가", "수정", "제거")
- 50자 이내

## 실행 순서

1. `git status`로 변경된 파일 목록 확인
2. `git diff`로 변경 내용 분석
3. 변경 내용을 바탕으로 적절한 type, scope, 설명 결정
4. 사용자에게 커밋 메시지 제안 후 승인 요청
5. 승인되면 아래 명령어 실행:

```bash
git add <변경된 파일들>  # git add . 금지 — 파일을 명시적으로 지정
git commit -m "type(scope): 설명"
git push
```

## 주의사항

- `.env`, `.env.local` 등 환경변수 파일은 절대 스테이징하지 마세요
- 여러 관심사가 섞인 경우 논리적 단위로 나눠서 커밋할지 사용자에게 물어보세요
- 커밋 전 반드시 사용자에게 메시지를 확인받으세요
